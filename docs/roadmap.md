# 3주 MVP 로드맵

## 목표

3주 안에 실제 업무에 쓸 수 있는 아임웹 제작 대행 운영 MVP를 배포한다.

핵심 사용자 흐름:

1. 잠재고객이 공개 사이트에서 포트폴리오, 가격, FAQ를 확인한다.
2. 제작 문의를 등록한다.
3. 관리자가 `/admin`에서 문의를 확인한다.
4. 문의를 고객과 프로젝트로 전환한다.
5. 계약금, 잔금, 추가 비용, 진행 상태를 관리한다.
6. AI로 문의 답변 초안, 제안서, 계약서 초안, 아임웹용 HTML/CSS/JavaScript를 생성한다.
7. 제작 완료 후 포트폴리오로 등록하고 공개한다.

## 1주차: 기반 구축과 핵심 운영 기능

목표: AI 기능을 제외한 기본 업무 흐름을 완성한다.

- Next.js, TypeScript, Tailwind, shadcn/ui 구성
- 공개 사이트와 관리자 레이아웃 분리
- Supabase 연결, 환경변수, 핵심 테이블, RLS 작성
- 관리자 로그인, 로그아웃, `/admin` 서버 측 접근 제한
- 공개 홈페이지: 홈, 서비스, 제작 과정, 가격, FAQ, 포트폴리오, 제작 문의 폼
- 관리자: 대시보드, 문의 목록/상세/상태 변경
- 문의에서 고객과 프로젝트 생성
- 고객, 프로젝트, 포트폴리오 기본 CRUD
- 계약금, 잔금, 추가 비용, 예상 매출, 확정 매출, 미수금 관리

완료 조건:

- 방문자가 실제 문의를 등록할 수 있다.
- 관리자만 문의를 확인할 수 있다.
- 문의를 고객과 프로젝트로 전환할 수 있다.
- 프로젝트 상태와 입금 현황을 관리할 수 있다.

## 2주차: AI 업무 보조와 아임웹 코드 생성

목표: 문의부터 제작 시작까지 반복 업무를 AI로 보조한다.

- AI API는 서버에서만 호출
- 모델 제공자 모듈과 프롬프트 템플릿 분리
- 요청, 응답, 모델, 토큰 사용량, 오류 기록
- AI 결과는 자동 발송하지 않고 초안으로 저장
- 문의 내용과 가격/FAQ 기반 답변 초안 생성
- 고객/프로젝트 정보 기반 제안서 초안 생성
- 계약서 초안 생성은 MVP 흐름에 포함하되 법적 자동 확정 문서가 아니라 편집 가능한 초안으로 제한
- 업종, 목적, 타깃, 분위기, 색상, 섹션, 인터랙션 기반 아임웹용 HTML/CSS/JavaScript 생성
- 생성 결과 저장, 복사, 재생성, 이전 버전 불러오기

완료 조건:

- 문의에서 AI 답변 초안을 만들 수 있다.
- 프로젝트에서 제안서와 계약서 초안을 만들 수 있다.
- 아임웹 코드 위젯에 붙여 넣을 코드를 생성, 저장, 복사할 수 있다.

## 3주차: 통합, 보안, QA, 배포

목표: 실제 업무에 사용할 수 있도록 주요 흐름을 검증하고 배포한다.

- 문의, 고객, 프로젝트 전체 검색
- 상태별 필터, 프로젝트 메모, 마감 임박 표시
- 최근 문의, 진행 프로젝트, 미수금 대시보드
- AI 생성 이력 조회
- 관리자 서버 측 인증 확인
- Supabase RLS 검증
- 공개 API와 관리자 API 분리
- AI API 키 클라이언트 노출 방지
- 개인정보와 토큰 로그 노출 방지
- Playwright로 주요 흐름 E2E 검증
- Vercel 배포, 도메인 연결, 운영 환경변수, 관리자 계정, 오류 로깅, 백업 방법 확인

완료 조건:

- 공개 사이트가 실제 배포되어 있다.
- 문의 등록부터 프로젝트/매출 관리까지 연결된다.
- AI 답변, 제안서, 계약서, 아임웹 코드 생성이 동작한다.
- 관리자 외에는 내부 데이터에 접근할 수 없다.
- 주요 테스트와 프로덕션 빌드가 통과한다.
